Scouter APM 설치 방법


    -javaagent:/usr/local/tomcat/lib/scouter.agent.jar
    -Dscouter.config=/usr/local/tomcat/conf/scouter.conf
    -Dobj_name={사용자 입력값 : 이름}
  • 에이전트 설정 방법
    • aRMS Version의 Scouter Version agent를 원하시는 Path에 위치시켜주세요
    • Tomcat start.sh 파일 혹은 env.sh 파일에 상기 코드를 삽입합니다.
    • Boot를 활용하는 경우에는 command에 삽입하여 명령을 실행합니다.
  • scouter.conf 설정 방법
    • obj_name={사용자 입력값 : 이름}
    • net_collector_ip={사용자 입력값 : 서버아이피}
    • net_collector_udp_port={사용자 입력값 : 포트}
    • net_collector_tcp_port={사용자 입력값 : 포트}
  • {사용자 입력값 : 이름}
    • Socuter Server 에서 모니터링 대상을 구분할 이름값 입니다.
    • 중복값은 데이터가 혼입될 수 있으니 주의하여 주세요

Elastic APM 설치 방법


    -javaagent:/usr/local/tomcat/lib/elastic-apm-agent.jar
    -Delastic.apm.service_name={사용자 입력값 : 이름}
    -Delastic.apm.application_packages={사용자 입력값 : 모니터링 대상 패키지}
    -Delastic.apm.server_urls={사용자 입력값 : 모니터링 서버 주소}
  • 에이전트 설정 방법
    • aRMS Version의 ELK APM Version agent를 원하시는 Path에 위치시켜주세요
    • 모니터링 할 서비스의 패키지를 설정합니다.
    • ELK APM 이 설치된 Docker Master Server 를 설정합니다.
    • Tomcat start.sh 파일 혹은 env.sh 파일에 상기 코드를 삽입합니다.
    • Boot를 활용하는 경우에는 command에 삽입하여 명령을 실행합니다.
  • {사용자 입력값 : 이름}
    • Elasticsearch Server 에서 모니터링 대상을 구분할 이름값 입니다.
    • 중복값은 데이터가 혼입될 수 있으니 주의하여 주세요
  • {사용자 입력값 : 모니터링 대상 패키지}
    • 서비스 코드 내부의 모니터링 할 대상 패키지를 설정합니다.
    • 대상 패키지는 다중으로 설정 할 수 있습니다.
    • 중복값은 데이터가 혼입될 수 있으니 주의하여 주세요
    • ex> egovframework.api,egovframework.com ...
  • {사용자 입력값 : 모니터링 서버 주소}
    • aRMS의 ELK APM은 Docker Swarm System으로 운용됩니다.
    • 따라서 설치된 Master Server에서 라우팅을 해 주기때문에
    • Docker Master Server와 Port로 구성된 주소를 설정 해 줍니다.
    • 중복값은 데이터가 혼입될 수 있으니 주의하여 주세요
    • ex> http://192.168.25.46:8200
